# 演示项目：闯关学习 + 签到积分

> **⚠️ 本项目为「AI 项目智能测试中心」的演示项目，非独立产品。**
>
> 本项目故意预埋了 6 个业务 Bug，用于演示测试中心的真实测试执行能力。
> 评委可通过测试中心（http://localhost:4000）一键导入此项目，体验完整的智能测试流程。

## 项目定位

一个闯关学习 + 签到积分的 Web 应用，用户通过答题闯关获取积分，每日签到领取奖励，积分可兑换礼品。

**技术栈**：Next.js 16 (App Router) + Prisma + SQLite + TypeScript + Tailwind CSS

## 预埋的 6 个业务 Bug

本项目故意预埋了 6 个"无技术报错但业务已崩"的隐藏 Bug，用于演示测试中心的主动发现能力：

| 编号 | Bug 描述 | 严重等级 | 发现阶段 |
|------|----------|----------|----------|
| Bug 1 | 无限签到：同一用户可无限次点击签到，反复领取积分 | CRITICAL | 高级业务测试 |
| Bug 2 | 双击重复：并发请求导致积分重复发放 | CRITICAL | 高级业务测试 |
| Bug 3 | 刷新可再签：刷新页面后签到状态丢失，可再次签到 | HIGH | 高级业务测试 |
| Bug 4 | 跳关：直接请求高关卡答题接口可绕过关卡顺序 | HIGH | 高级业务测试 |
| Bug 5 | 进度丢失：答题完成后刷新，关卡状态未持久化 | MEDIUM | 基础测试 |
| Bug 6 | 积分不足兑换：0 积分也能成功兑换礼品 | CRITICAL | 高级业务测试 |

## 启动方式

```bash
# 端口 4010 为固定值，不可更改（详见根目录 PORT_RULES.md）
npm install
npm run dev
```

启动后访问 http://localhost:4010

## 与测试中心的配合

1. 启动本项目（端口 4010）
2. 启动测试中心（端口 4000）
3. 在测试中心点击"一键导入演示项目"
4. 选择"真实执行模式"体验完整的智能测试流程
5. 测试中心将真实操作浏览器、调用 API、读取数据库，发现上述 6 个 Bug

## 数据库

使用 SQLite（`prisma/dev.db`），首次运行需执行：

```bash
npx prisma db push
```
